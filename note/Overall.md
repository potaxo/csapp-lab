## Style Guide
- Documentation
  - function -> arguments, return value, error cases, pertinent side effects. [Example](./detailed_explaination_of_documentation.md#function-header-example)
  - whitespace
  - line length `wc -L file.c`
  - variable naming
  - Magic number `#define BUFLEN 256`
  - No dead code
  - Modularity
  - Failure condition handling
  - Memory and files management
  - Consistent formatting
